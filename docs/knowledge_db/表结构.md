## Contents 表 (内容数据库)

Contents 表存储知识条目的元数据信息，位于关系型数据库中。

### PostgreSQL 表结构

**表名**: `agno_knowledge`

| 字段名 | 类型 | 约束 | 说明 |
|---------|------|--------|------|
| `id` | String | PRIMARY KEY, NOT NULL | 知识条目的唯一标识符 |
| `name` | String | NOT NULL | 知识条目的名称 |
| `description` | Text | NOT NULL | 知识条目的详细描述 |
| `metadata` | JSONB | NULLABLE | 元数据，以 JSONB 格式存储 |
| `type` | String | NULLABLE | 知识类型 |
| `size` | BigInteger | NULLABLE | 内容大小（字节） |
| `linked_to` | String | NULLABLE | 关联标识符，用于多个 Knowledge 实例共享同一向量数据库时的隔离 |
| `access_count` | BigInteger | NULLABLE | 访问计数 |
| `status` | String | NULLABLE | 状态 |
| `status_message` | Text | NULLABLE | 状态消息 |
| `created_at` | BigInteger | NULLABLE | 创建时间戳（Unix 时间戳） |
| `updated_at` | BigInteger | NULLABLE | 更新时间戳（Unix 时间戳） |
| `external_id` | String | NULLABLE | 外部 ID |

## PgVector 向量表

PgVector 向量表存储文档的向量嵌入和内容，使用 PostgreSQL 的 pgvector 扩展。

### 表结构定义

**表名**: 由用户指定（默认在 `ai` schema 下）

| 字段名 | 类型 | 约束 | 说明 |
|---------|------|--------|------|
| `id` | String | PRIMARY KEY | 文档的唯一标识符（包含 content_hash） |
| `name` | String | NULLABLE | 文档名称 |
| `meta_data` | JSONB | DEFAULT '{}' | 文档元数据 |
| `filters` | JSONB | NULLABLE | 过滤器条件 |
| `content` | Text | NULLABLE | 文档内容 |
| `embedding` | Vector(dimensions) | NULLABLE | 向量嵌入 |
| `usage` | JSONB | NULLABLE | API 使用情况 |
| `created_at` | DateTime(timezone=True) | DEFAULT now() | 创建时间 |
| `updated_at` | DateTime(timezone=True) | ON UPDATE now() | 更新时间 |
| `content_hash` | String | NULLABLE | 内容哈希值 |
| `content_id` | String | NULLABLE | 内容 ID |

### 索引

| 索引名 | 类型 | 字段 | 说明 |
|---------|------|------|------|
| `idx_{table_name}_id` | B-tree | `id` | 主键索引 |
| `idx_{table_name}_name` | B-tree | `name` | 名称索引 |
| `idx_{table_name}_content_hash` | B-tree | `content_hash` | 内容哈希索引 |
| `idx_{table_name}_content_id` | B-tree | `content_id` | 内容 ID 索引 |
| `{table_name}_{index_type}_index` | IVFFlat/HNSW | `embedding` | 向量相似性索引 |

### 表结构 SQL（Schema V1）

```sql
CREATE TABLE {table_name} (
    id VARCHAR PRIMARY KEY,
    name VARCHAR,
    meta_data JSONB DEFAULT '{}',
    filters JSONB,
    content TEXT,
    embedding vector({dimensions}),
    usage JSONB,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT now(),
    updated_at TIMESTAMP WITH TIME ZONE,
    content_hash VARCHAR,
    content_id VARCHAR
);
```

### 向量索引类型

#### HNSW (Hierarchical Navigable Small World)

```sql
CREATE INDEX "{index_name}" ON {table_name}
USING hnsw (embedding {distance_metric})
WITH (m = :m, ef_construction = :ef_construction);
```

#### IVFFlat

```sql
CREATE INDEX "{index_name}" ON {table_name}
USING ivfflat (embedding {distance_metric})
WITH (lists = :num_lists);
```

### GIN 全文搜索索引

```sql
CREATE INDEX "{gin_index_name}" ON {table_name}
USING GIN (to_tsvector('{language}', content));
```

### 距离度量

| 度量 | PostgreSQL 操作符 | 说明 |
|------|-----------------|------|
| Cosine | `vector_cosine_ops` | 余弦距离 |
| L2 | `vector_l2_ops` | 欧几里得距离 |
| Max Inner Product | `vector_ip_ops` | 最大内积 |

## ClickHouse 向量表

ClickHouse 向量表使用 `ReplacingMergeTree` 引擎和向量相似性索引。

### 表结构定义

**表名**: 由用户指定

| 字段名 | 类型 | 约束 | 说明 |
|---------|------|--------|------|
| `id` | String | PRIMARY KEY | 文档的唯一标识符 |
| `name` | String | NULLABLE | 文档名称 |
| `meta_data` | JSON | DEFAULT '{}' | 文档元数据 |
| `filters` | JSON | DEFAULT '{}' | 过滤器条件 |
| `content` | String | NULLABLE | 文档内容 |
| `content_id` | String | NULLABLE | 内容 ID |
| `embedding` | Array(Float32) | NULLABLE | 向量嵌入 |
| `usage` | JSON | NULLABLE | API 使用情况 |
| `created_at` | DateTime('UTC') | DEFAULT now() | 创建时间 |
| `content_hash` | String | NULLABLE | 内容哈希值 |

### 表结构 SQL

```sql
CREATE TABLE IF NOT EXISTS {database_name}.{table_name}
(
    id String,
    name String,
    meta_data JSON DEFAULT '{}',
    filters JSON DEFAULT '{}',
    content String,
    content_id String,
    embedding Array(Float32),
    usage JSON,
    created_at DateTime('UTC') DEFAULT now(),
    content_hash String,
    PRIMARY KEY (id),
    INDEX embedding_index embedding TYPE vector_similarity('hnsw', 'L2Distance', {dimensions}, {quantization}, {hnsw_max_connections_per_layer}, {hnsw_candidate_list_size_for_construction})
) ENGINE = ReplacingMergeTree ORDER BY id
```

## 索引和优化

### PgVector 优化

1. **向量索引**: 使用 HNSW 或 IVFFlat 加速向量相似性搜索
2. **GIN 索引**: 加速全文本搜索
3. **B-tree 索引**: 加速基于 `id`、`name`、`content_hash`、`content_id` 的查询

### ClickHouse 优化

1. **向量相似性索引**: 使用 HNSW 算法进行向量搜索
2. **ReplacingMergeTree**: 自动去重和合并
