# 桌面操控 Agent 需求文档

## 1. 项目概述

### 1.1 需求背景
GPT-5.4 新增了内置 Computer use 能力，模型可以通过查看屏幕截图并执行点击、输入、滚动等UI操作来操控软件。项目中已有基于 Playwright 的 `browser_use_agent` 和 `web_driver_monitor` 服务，但尚未利用模型原生的 Computer use 工具能力。

### 1.2 目标
创建一个桌面操控 Agent，能够：
- 通过屏幕视觉理解操作本地桌面或浏览器界面
- 执行点击、输入、滚动等UI交互
- 完成多步骤自动化任务流程
- 支持代码执行模式处理复杂逻辑

---

## 2. 核心功能需求

### 2.1 屏幕视觉交互
**描述**：Agent 通过截图感知桌面状态，执行标准UI操作。

| 功能 | 说明 |
|------|------|
| 截图查看 | 获取当前屏幕/窗口截图供模型分析 |
| 点击 | 单击指定坐标位置，支持左键/右键 |
| 双击 | 双击指定坐标位置 |
| 滚动 | 在指定位置执行滚动操作 |
| 文本输入 | 在当前焦点位置输入文本 |
| 按键操作 | 发送键盘快捷键或特殊键 |
| 拖拽 | 按下并拖动到目标位置 |
| 鼠标移动 | 移动鼠标指针到指定位置 |
| 等待 | 延迟等待界面变化 |

### 2.2 多环境支持
**描述**：支持浏览器和本地桌面两种操控环境。

| 环境类型 | 说明 |
|-----------|------|
| 浏览器环境 | 操控网页表单、导航、交互 |
| 本地桌面环境 | 操控本地应用（如Office、IDE等） |
| 应用窗口管理 | 指定特定应用窗口进行操作 |

### 2.3 任务编排与记忆
**描述**：支持复杂多步骤任务的分解与状态记忆。

| 功能 | 说明 |
|------|------|
| 任务分解 | 将用户任务分解为多个可执行的UI操作 |
| 状态追踪 | 记录当前操作步骤和任务进度 |
| 错误恢复 | 操作失败后尝试恢复或调整策略 |
| 上下文保持 | 保持浏览器/应用状态跨轮次持续 |

### 2.4 代码执行模式
**描述**：支持 Agent 编写和执行代码以完成复杂任务。

| 功能 | 说明 |
|------|------|
| 脚本编写 | 模型可编写 Playwright/PyAutoGUI 脚本 |
| 环境持久化 | 保持浏览器/运行时对象存活跨调用 |
| DOM操作 | 支持基于DOM的程序化交互 |
| 输出控制 | 控制文本输出和截图返回给模型 |

### 2.5 安全与确认机制
**描述**：对高风险操作进行人工确认。

| 确认级别 | 风险操作类型 |
|----------|-------------|
| **必须接管** | 修改密码、绕过安全屏障、CAPTCHA |
| **必须确认** | 删除数据、修改权限、发送敏感信息、金融交易 |
| **预批准即可** | 登录、上传文件、接受权限提示 |
| **无需确认** | 安全范围内的常规操作 |

## 3. 技术架构设计

### 3.1 组件划分

```
┌─────────────────────────────────────────────────────────────────┐
│                        AgentOS 主应用                           │
│                       (api/main.py)                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                  desktop_control_agent                        │
│            (agent/desktop_control_agent.py)                   │
└─────────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
┌─────────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│ ComputerMCP        │ │ 现有 WDM      │ │ CodeExecMode   │
│  (server/         │ │  (复用)        │ │  (可选扩展)    │
│   computer_mcp/)    │ │                 │ │               │
└─────────────────────┘ └─────────────────┘ └─────────────────┘
```

### 3.2 实现路径选择

| 路径 | 适用场景 | 复用建议 |
|------|----------|-----------|
| **原生Computer工具** | 标准UI交互、网页导航 | 推荐优先使用 |
| **自定义Harness** | 已有Playwright基础设施 | 复用web_driver_monitor |
| **代码执行模式** | 复杂逻辑、DOM操作 | 可作为高级扩展 |

---

## 4. 集成策略

### 4.1 复用现有能力
- 复用 `server/web_driver_monitor` 的 Playwright 基础设施
- 复用现有浏览器会话管理机制
- 复用 Prometheus 监控端点

### 4.2 新增模块
- 创建 `server/computer_mcp/` 作为 Computer use 执行层
- 创建 `agent/desktop_control_agent.py` 使用模型原生computer工具
- 创建配置文件管理操作环境（分辨率、允许域名等）

---

## 5. 配置项

| 环境变量 | 默认值 | 说明 |
|----------|--------|------|
| COMPUTER_MCP_URL | http://computer-mcp:8013/mcp | Computer MCP 服务地址 |
| DESKTOP_RESOLUTION | 1440x900 | 桌面分辨率 |
| BROWSER_HEADLESS | false | 是否无头模式 |
| ALLOWED_DOMAINS | * | 允许访问的域名 |
| CONFIRMATION_LEVEL | default | 确认级别 |

---

## 6. 功能优先级

### Phase 1: 基础能力
1. 使用模型原生 computer 工具实现浏览器操控
2. 实现截图与基础动作执行
3. 集成到 AgentOS

### Phase 2: 安全增强
1. 实现分级确认机制
2. 实现域名白名单
3. 实现敏感数据检测

### Phase 3: 高级功能
1. 实现代码执行模式
2. 实现桌面应用操控
3. 实现任务模板与录制

---

## 7. 风险与限制

| 风险 | 缓解措施 |
|------|----------|
| 误操作 | 高风险操作强制人工确认 |
| 敏感信息泄露 | 检测并阻止敏感数据输入 |
| 无限循环 | 设置最大步数限制 |
| 资源占用 | 会话自动清理、超时控制 |
